# ケアプラン作成支援システム 利用者マニュアル

## はじめに

本システムは、ケアマネジャーが居宅サービス計画書（第1表〜第3表）を効率的に作成するためのWebアプリケーションです。Googleドライブに蓄積された利用者情報をAIが分析し、複数のケアプラン案を提案します。

---

## 1. ログイン

1. ブラウザでシステムのURLにアクセスします
2. 「Googleでログイン」ボタンをクリックします
3. Googleアカウントを選択し、必要な権限（ドライブ・スプレッドシート）を許可します
4. ログインが完了するとホーム画面が表示されます

> **注意**: 管理者が許可リストに登録したメールアドレスのみログインできます。

---

## 2. ホーム画面

ホーム画面では以下の操作ができます:

- **新規ケアプラン作成**: 青いカードをクリックして作成フローを開始
- **最近のエクスポート**: 過去にエクスポートしたケアプランの履歴を確認
- **設定**: 右上の「設定」ボタンから事業所情報やプロンプトを管理

---

## 3. ケアプラン作成フロー

### ステップ1: 利用者選択

1. Googleドライブの利用者フォルダ一覧が表示されます
2. 検索ボックスで名前を絞り込めます
3. 対象の利用者をクリックして選択します
4. 「次へ」ボタンで情報源選択に進みます

### ステップ2: 情報源選択

1. **事業形態を選択**します:
   - 居宅介護支援（通常）
   - 小規模多機能型居宅介護
2. 利用者フォルダから見つかったファイルが一覧表示されます
3. 各カテゴリの最新ファイルが自動選択されています
4. 必要に応じてチェックを付け外しします
5. 🔒マークは機密文書（マイドライブ配下）を示します
6. 「AI分析を実行」ボタンで分析を開始します

**ファイルカテゴリ**:
| アイコン | カテゴリ |
|---------|---------|
| 📋 | 既存ケアプラン |
| 🏥 | 主治医意見書 |
| 📊 | 認定調査票 |
| 👥 | 担当者会議録 |
| 📝 | アセスメント |
| 📒 | 実施記録 |
| 🗂 | フェイスシート |

### ステップ3: プラン編集・確認

AI分析完了後、3つのプラン案が生成されます:

1. **プラン切り替え**: 上部のボタンで各プラン案を比較
   - 自立支援重視プラン（青）
   - 安全・安心重視プラン（緑）
   - バランス型プラン（橙）
   - 既存プラン（紫）※既存プランを情報源に含めた場合

2. **表の切り替え**: タブで第1表・第2表・第3表を切り替え

3. **編集**: 第1表のテキスト欄は直接編集可能です

4. **操作ボタン**:
   - **下書き保存**: 現在の状態を保存（後で再開可能）
   - **Googleスプレッドシートにエクスポート**: 即座にエクスポート
   - **エクスポート画面へ**: 詳細なエクスポートオプション画面に遷移

### ステップ4: エクスポート

- **Googleスプレッドシート**: 利用者フォルダの`01_居宅サービス計画書/`配下にスプレッドシートを作成
- エクスポート完了後、リンクが表示されスプレッドシートを直接開けます

---

## 4. 設定画面（管理者向け）

### 基本情報

- 事業所名・所在地・管理者名
- Google Drive 連携フォルダID
- AIモデル選択
- 提案数設定

### プロンプト管理

6種類のプロンプトを個別に編集できます:
- 居宅介護支援: 第1表・第2表・第3表
- 小規模多機能: 第1表・第2表・第3表

**使用可能な変数**:
`{利用者名}` `{要介護度}` `{生年月日}` `{住所}` `{アセスメント情報}` `{既存ケアプラン}` `{通い記録}` `{訪問記録}` `{泊まり記録}` `{主治医意見書}` `{担当者会議録}` `{フェイスシート}` `{事業所名}` `{管理者名}`

### 許可リスト

ログインを許可するGoogleアカウントのメールアドレスを管理します。
- `admin`: 設定画面にアクセス可能
- `user`: ケアプラン作成のみ

---

## 5. トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| ログインできない | 許可リストにメールアドレスが登録されているか確認 |
| 利用者が表示されない | 設定画面でフォルダルートIDが正しく設定されているか確認 |
| ファイルが見つからない | ドライブ上のフォルダ構造がAutofiler形式に従っているか確認 |
| AI分析がエラーになる | Gemini APIキーの有効性と利用制限を確認 |
| エクスポートが失敗する | ドライブへの書き込み権限があるか確認 |

---

## 6. セキュリティに関する注意

- 利用者の個人情報がAI（Gemini API）に送信されます
- 機密文書は実行ユーザーのマイドライブにのみ保存されます
- ログアウト後はセッション情報が破棄されます
- 他のユーザーのマイドライブの機密文書にはアクセスできません
